PART 1 — CORA CHAT ADMIN PANEL
==================================================

Create a lightweight internal admin UI for Cora.

Suggested route:
- /admin/cora
or the most consistent admin route already used in the app

This should be an INTERNAL admin surface only.

Core layout:
- left column: conversation / requests / recent plans
- main panel: Cora chat thread
- right panel (or tabs): plan preview / outputs / suggestions / questions

Minimum chat capabilities:
1. enter freeform request
2. submit to existing Cora service
3. display:
   - active hat
   - message
   - structured outputs
   - whether approval is required
4. support response cards for:
   - plan preview
   - source selections
   - content outputs
   - outreach drafts
   - UI proposals

Do NOT over-design this.
Keep it practical, admin-first, and readable.

PART 4 — VISUAL / UI CHANGE PROPOSALS
==================================================

Create a lightweight UI proposal system.

ui_change_proposals
- id
- scope (platform / metro)
- metro_id (nullable)
- name
- change_type (colors, layout, typography, component, page)
- description
- preview_config (json)
- code_snippet (text, nullable)
- status (draft, approved, applied, reverted)
- created_by
- created_at
- updated_at

Cora should support requests like:
- "Change the colors"
- "Make this page cleaner"
- "Update the homepage look"
- "Adjust cards / spacing / header"

Behavior:
- Hat should resolve to Builder or CTO (visual mode)
- Cora must respond with:
  - what is changing
  - why
  - preview_config
  - code snippet or implementation notes if relevant
- Save as draft proposal
- Allow:
  - Approve
  - Edit
  - Copy code/prompt
  - Revert later if applied manually

IMPORTANT:
This is proposal-first only.
Do NOT directly change UI automatically.
