GOAL
Implement an RSS ingestion pipeline as part of our multi-metro “Metro Sources” framework, starting with Charlotte. Automatically pull RSS feeds (hourly), normalize items, store them, and render a simple “Seed Content” module on the front end (homepage + neighborhood pages optional) WITHOUT turning the site into a static directory. This is for lightweight, rotating community content.

CONSTRAINTS
- No manual entry.
- Use only public RSS URLs.
- Respect source links: always link back to original article URL; do not copy full articles.
- Store only: title, link, published date, short summary/description if present, source name, optional image if present.
- Must work for multiple metros via metro_sources config.
- No paywalls circumvention; if the article is paywalled, we still show title + link only.
