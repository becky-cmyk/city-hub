BUSINESS RULES TO IMPLEMENT

1. SINGLE SOURCE / OWNERSHIP
- Every listing must have one owner record.
- Owner can be:
  a) hub_presence_id
  or
  b) reader_user_id
- Add clear owner type handling.
- Do not allow ambiguous ownership.

2. READER LISTING RULES
- Reader-owned classifieds are allowed.
- Reader-owned listings require verified account status before publishing.
- Reader-owned listings are capped at 3 active listings total across all marketplace categories unless later overridden by admin.
- This cap must be enforced at create + publish time.
- Reader-owned FSBO listings are allowed only if verified.
- Reader-owned housing demand listings (looking for rental / roommate) are allowed only if verified.

3. HUB PRESENCE RULES
- Hub Presence owners can create listings as business/organization/property operators.
- Preserve the existing add-on / capability model so listing-heavy usage can later be gated by add-on, quotas, credits, and visibility boosts.
- Do not hardcode unlimited unless the current entitlement model already supports it cleanly.
- Structure the code so limits can be configured by tier later.

4. FSBO RULE
- Add explicit subtype/tag for FOR SALE BY OWNER.
- FSBO lives in Marketplace, not MLS.
- FSBO can belong to verified readers.
- FSBO should be searchable/filterable distinctly.

5. MLS RULE
- Do NOT ingest MLS into the marketplace database.
- Marketplace housing supply is for internal/listed objects only.
- Keep MLS separate for future sponsor embed blocks on relocation or hub pages.
- Do not mix external MLS records into this schema.

6. HOUSING DEMAND RULE
- Add support for “Looking for Rental,” “Looking for Roommate,” and “Housing Wanted.”
- These are demand-side marketplace objects.
- They must support budget, target area, move-in timing, and preferences.
- They should appear in search and marketplace pages, and be optionally eligible for Pulse story-style surfacing later.

7. COMMERCIAL PROPERTY RULE
- Commercial listings live in Marketplace and must be their own major category.
- Must support lease vs sale distinction.
- Must support property type and square footage fields.
- Must be usable later in business relocation and “open a business here” flows.

8. SERVICES RULE
- Services are marketplace listings, not a separate architecture.
- Services can later route users into Hub Presence upgrades.
- Must support category + service area + pricing style.

9. FOR SALE RULE
- General local items can be posted, but keep structure clean.
- Add expiration and moderation support to avoid clutter / stale inventory.

10. COMMUNITY RULE
- Community / opportunity listings should support simple civic or local-activity use cases.
- These should remain structured, moderateable, and geo-aware.
