PART 1 — BUILD CHARLOTTE ORCHESTRATOR
--------------------------------------------------

Create a central orchestration service/module for Charlotte.

Suggested name:
- charlotte-orchestrator.ts
or equivalent

Purpose:
- accept natural language requests or structured directives
- classify intent
- identify entities involved
- determine the correct engine(s)
- either:
  A. propose an action plan
  B. execute allowed actions directly
- log actions and outcomes

The orchestrator must support these broad operating modes:

1. OPERATOR MODE
Examples:
- “Start a TV venue for ABC Coffee in Pineville”
- “Create a story for these expo leads”
- “Set this up as a radio partner”

2. PROPOSAL MODE
Examples:
- “What should we do with these?”
- “Process these and tell me what actions make sense”

3. SEARCH / DISCOVERY MODE
Examples:
- “Best brunch in Ballantyne”
- “Who should I talk to for a mortgage in Pineville?”
- “What are good things to do this weekend?”

4. CONCIERGE / NAVIGATION MODE
Examples:
- housing
- jobs
- services
- events
- marketplace guidance
- relocation-style flows

5. BRAINSTORM / CREATIVE MODE
Examples:
- “Give me newsletter ideas for girls night out”
- “What content should we create this week?”

Do NOT make these five separate AI systems.
Use one orchestrator with mode detection and routing.

PART 2 — BUILD INTENT PARSER / COMMAND ROUTER
--------------------------------------------------

Create an intent parser that converts natural language into structured internal commands.

For each request, identify at minimum:
- mode
- primary intent
- target entity/entities
- target engine(s)
- location / geo context
- desired action(s)
- whether proposal is required first
- confidence

Example:

Input:
“Start a TV venue for ABC Coffee on Main Street in Pineville”

Should resolve approximately to:
- mode: operator
- entity_type: business
- entity_name: ABC Coffee
- geo: Pineville
- action: ACTIVATE_TV_VENUE
- engines: Hub Presence + media network + CRM note/log
- proposal_required: false or true depending on confidence / missing data

Support both:
- single-item commands
- multi-item/batch commands

  A. hub/master radio participation

9. CROWN_CANDIDATE_TEMPLATE

--------------------------------------------------

Charlotte must support:
“do this to all of these”

- grouping uploads/captures into a batch
- attaching a shared directive
- applying default actions to all
- applying item-level overrides

- “These are expo leads — write stories, reach out, and schedule”
- “These are advertiser opportunities — identify contact paths and generate conversion outreach”

1. ingest items
2. resolve entities
3. classify batch type when possible
4. build proposal
5. allow confirm/execute
6. track per-item status

Support mixed results gracefully.
