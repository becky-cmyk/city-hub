GOAL
Upgrade the existing Hub Screens / TV system so the back office can manage:
- full 60-minute “program loops”
- optional expansion to 90-minute programming
- mixing of multiple full loops across the day
- dynamic interchanging of loops so the channel feels alive, not repetitive
- voiceover / hosted narration segments
- subtitles / closed captions
- venue-specific inserts
- metro + micro hub + location layering
- competitor protection rules
- monetization inventory that feels like “Support Local / Community Spotlight” rather than only ads

IMPORTANT CONTEXT
This is NOT just static signage.
This should feel like a real local channel for the area and the venue.

The venue owner/staff should NOT hear the exact same content in the same order all day.
We want:
- full one-hour loops
- multiple loops
- loops mixed and rotated
- some spoken segments
- some silent/visual slides
- some video clips
- captions/subtitles for spoken/video content
- venue personalization
- local community feel

PROGRAMMING MODEL TO IMPLEMENT
We are moving from a simple 20-minute repeating loop to a “programmed channel” model.

A. FULL LOOPS
Add support for full loops:
- 60-minute loops as the standard
- optional 90-minute loops supported by the system
- each loop has a name, theme, target audience, target venue types, and item sequence

Examples:
- Local Pulse Hour
- Community Spotlight Hour
- Tonight Around You Hour
- Venue + Neighborhood Hour
- Info + Weather + Traffic + Sports Hour
- Weekend / Nightlife Hour

B. LOOP MIXING
The system must support:
- assigning multiple full loops to a screen
- rotating those loops throughout the day
- randomizing the order within allowed scheduling rules
- mixing in venue inserts regardless of which loop is running
- mixing in real-time/current items (weather, traffic, sports) without rebuilding the entire loop

C. CHANNEL FEEL
The final result must feel like:
- a live local community channel
- not static signage
- not the same repeated sequence
- venue-aware and neighborhood-aware
- partially hosted via narration

D. AUDIO / NARRATION
Support “spoken segments” for selected templates.
This means some templates can become short narrated video segments instead of just slides.

Examples of spoken segments:
- Neighborhood Spotlight
- Community / Nonprofit Spotlight
- Support Local / Local Commerce
- Tonight Around You
- Weekend Happenings
- Hub Discovery
- QR CTA intro
- Event intro

Voiceover should sound like a local host introducing content.

Examples of style:
- “We’d like to introduce you to one of the local businesses helping shape this neighborhood…”
- “Here’s something happening around you this weekend…”
- “Let’s take a quick look at what’s happening nearby tonight…”

Audio must be optional per item/template.
Not every item should talk.
We want spoken segments spaced so they do not fatigue staff or venue owners.

Recommended rule:
- spoken segments every 3–6 minutes on average, configurable per loop/screen

E. CAPTIONS / SUBTITLES
For all spoken segments and videos, support captions/subtitles.
Requirements:
- WebVTT support
- optional SRT support if easy
- captions auto-display when available
- large, readable caption styling for TVs
- captions on by default for venue screens
- if the item has narration text, support auto-generation of subtitles from narration text even without a separate caption upload
- spoken segments rendered from slide templates must be able to display synchronized subtitle text

F. VENUE / MICRO / METRO LAYERING
Each screen feed should still support content coming from:
- Metro
- Micro Hub
- Location/Venue

Example:
- metro-wide event / weather / sports / support local
- micro hub neighborhood items
- location-specific specials and promotions

G. COMPETITOR PROTECTION
If a venue hosts the screen:
- protect against direct competitor placements
- example: gelato shop should not show another gelato shop
- use category-based exclusion and explicit excluded business lists
- venue content should always be eligible
- “Support Local” placements should respect protection rules

H. MONETIZATION / INVENTORY FRAMING
Do NOT make all monetized content feel like generic ads.
We want inventory types such as:
- Support Local
- Local Commerce Spotlight
- Community Partner
- Neighborhood Favorite
- Nonprofit Showcase
- Around the Neighborhood
- Tonight Around You
- Weekend Happenings

Some of those may still be paid placements, but the framing should feel editorial/community-oriented.

DO NOT build billing right now.
Only build management structures for placements and scheduling.
