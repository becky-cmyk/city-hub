07 — SEO + AUTHORITY ENGINE (NEWLY CONFIRMED / SOURCE OF TRUTH)
A) SEO Diagnostic (verified working)
- Token-protected public endpoint works and is secure:
  - Wrong/missing token -> 403
  - Correct token -> full diagnostic JSON
- Dev “fetch failed” fixed by base URL resolver:
  - APP_PUBLIC_URL (prod) else REPLIT_DEV_DOMAIN else http://127.0.0.1:<PORT>

B) Critical finding (resolved)
- App was fully client-rendered; raw HTML was identical SPA shell with empty tags.
- This was blocking real crawl/index visibility.

C) Solution implemented (verified)
SEO “Snapshot Injection” for public routes:
- Server builds meaningful HTML per route using DB data
- Injects into <head>:
  - Title
  - Meta description
  - Canonical
  - JSON-LD
- Injects into <div id="root">:
  - H1 + minimal visible content + real internal links
- SPA scripts still load normally for users (additive for crawlers)

D) Snapshot safety (verified)
- Snapshot wrapped in #seo-snapshot
- Inline JS adds html.js class; CSS hides snapshot for JS-enabled browsers
- Prevents double-render flash; keeps snapshot visible for non-JS crawlers
- Head injection is idempotent (no duplicate meta/canonical/jsonld)

E) Authority clusters added (verified)
- Internal link modules on snapshot pages:
  - Explore Categories
  - Explore Zones
  - Featured Businesses
  - “More in Category / More in Zone” (business detail; depends on seed data)
- JSON-LD improvements:
  - BreadcrumbList on all snapshot pages
  - ItemList on list pages (directory/category/zone)

F) Current SEO state (verified pass)
- Public pages now have:
  - Non-empty <title>
  - Meta description present
  - Canonical present
  - JSON-LD present (LocalBusiness/Breadcrumb/ItemList as appropriate)
  - H1 present
  - Content checks true (business/category/zone names present in raw HTML)
  - Strong internal linking (20+ links on list pages)
