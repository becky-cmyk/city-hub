You are working inside the existing CLT Metro Hub / Charlotte City Hub codebase.

GOAL
Add “Web TV” (venue screens) + a back-office admin area to manage screens, content items (slides + short videos), scheduling/rotation rules, and monetization placements.
This must support the hierarchy:
- Metro Hub (citywide)
- Micro Hub (neighborhood/niche)
- Location (specific venue hosting a screen)
Revenue shares are future-proofed for: Venue, Micro Operator, Metro Operator, Core.
Right now only Venue share is active; all other shares roll up to Core automatically.

NON-NEGOTIABLES
- Keep it lightweight: mostly slides (7–12 seconds) + occasional short video clips (30–60 seconds).
- 20-minute loop target (approximate; dynamic playlist is ok).
- Glanceable templates: 1 idea per slide, large text, minimal copy, always include QR when appropriate.
- Must allow competitor blocking (e.g., host gelato shop should not show other gelato shops).
- No new paid services required. Use existing DB + storage approach in the app (or local assets if that’s what exists). If storage is needed, implement abstraction with pluggable provider.

DELIVERABLES
A) PUBLIC “TV” ROUTES
1. /tv (full-screen player)
   - Accepts query params:
     - metroSlug (optional; default current city/metro)
     - hubSlug (micro hub slug, required for micro feeds)
     - locationSlug (optional; for venue-specific inserts)
     - mode=autoplay (default)
   - Example:
     - /tv?hubSlug=southend
     - /tv?hubSlug=southend&locationSlug=ginmill
   - Must run fullscreen, kiosk-friendly, no nav chrome.
   - Pull playlist from API and play continuously.
   - Should refresh playlist every 5 minutes without breaking playback.

2. /api/tv/playlist
   - Inputs: metroSlug, hubSlug, locationSlug
   - Output: ordered list of “TV items” with timing/duration and render instructions:
     {
       items: [
         { id, type: "slide"|"video", durationSec, templateKey, data, assetUrl, clickUrl, qrUrl, priority, source: "metro"|"hub"|"location" }
       ],
       generatedAt,
       nextRefreshSec
     }
   - Playlist generation rules:
     - Default mix target:
       - 70–80% slides (7–12s each)
       - 20–30% short videos (30–60s each)
     - Loop length target ~20 minutes (allow slight variance).
     - Always include venue promo slides if locationSlug exists (frequency: every 2–4 minutes).
     - Insert “Hub discovery” slide at least once per loop.
     - If no content exists for hub/location, gracefully fall back to metro content.
