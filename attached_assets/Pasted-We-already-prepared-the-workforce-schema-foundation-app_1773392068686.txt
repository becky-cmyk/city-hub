--------------------------------------------------
2. ENFORCE ROLE-SPECIFIC LISTINGS
--------------------------------------------------

Build validation rules so employers cannot create broad catch-all listings.

Examples that should be blocked or flagged:
- “Servers / Bartenders / Hosts”
- “Hiring All Positions”
- “Now Hiring 100 People”
- “Office / Sales / Customer Support”

System should require:
- one title per listing
- one role category per listing
- positionsAvailable can be more than 1, but only for the same role

If the title appears to contain multiple roles:
- warn the employer
- require correction before publishing

Create a helper function/service to detect likely mixed-role titles.

--------------------------------------------------
3. DUPLICATE LISTING PREVENTION
--------------------------------------------------

Implement duplicate-prevention logic.

A business should generally have only one active listing per:
- normalizedJobTitle
- same location
- same employmentType
- same employerType

If a likely duplicate exists:
show warning:
“You already have a similar active listing. Update the existing listing instead of creating a duplicate.”

Actions:
- open existing listing
- edit existing listing
- continue with explanation (admin review flag)

Duplicate detection should consider:
- same business
- same normalized title
- same location
- same category
- high description similarity

Do NOT allow job-feed inflation by reposting the same listing as a new listing.

--------------------------------------------------
4. LISTING RENEWAL LOGIC
--------------------------------------------------

Renewals must update the existing listing history, not create fake duplicates in the feed.

Create a renewal workflow with required employer explanation.

Renewal reason options:
- we hired someone and need to hire another person
- we are expanding the team
- we are still trying to fill the same opening
- we did not find a qualified candidate
- hiring timeline changed

Logic:
A. If employer says:
- we hired someone and need to hire another person
- we are expanding the team

Then:
- renewal pricing stage stays at base level
- renewal penalty counter resets or does not escalate
- hiresCount can be incremented
- listing remains valid

B. If employer says:
- still trying to fill the same opening
- did not find a qualified candidate

Then:
- renewalCount increments
- escalating renewal pricing stage applies later
- listing integrity score should be impacted

Track:
- number of hires made from this listing
- positions originally requested
- positions still open
