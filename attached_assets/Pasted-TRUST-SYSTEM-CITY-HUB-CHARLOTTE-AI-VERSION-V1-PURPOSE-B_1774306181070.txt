TRUST SYSTEM — CITY HUB / CHARLOTTE AI
VERSION: V1
PURPOSE: Build the Trust System layer that sits on top of existing Hub Presence, Story Builder, reviews, badges, and recommendation logic.

IMPORTANT:
- Do not replace existing verification, Story Builder, Ask an Expert, Crown, review, or badge logic.
- Build this as a unifying trust layer that organizes, scores, displays, and enforces trust signals already present in the platform.
- Preserve existing architecture: Hub Presence = identity layer, objects = created once, surfaced across multiple discovery surfaces.
- No duplicate trust records disconnected from Hub Presence.
- No single public “trust score” shown to users.
- Public trust should be shown as stacked signals / badges / statuses, not one numeric score.
- Keep stars for reviews. Do not remove star ratings.
- Add structured trust layers around them.

==================================================
A. GOAL
==================================================

Build a TRUST SYSTEM that:
1. Organizes all visible and internal trust signals
2. Supports public trust display on businesses / creators / experts / organizations
3. Feeds Charlotte recommendations
4. Supports Trusted Network eligibility
5. Tracks trust decay / needs-attention / at-risk states
6. Allows recovery, not just punishment
7. Works across micro hubs / neighborhoods / community contexts

This Trust System must support:
- general businesses
- restaurants
- service providers
- creators
- experts / Ask an Expert
- nonprofits / organizations
- event organizers
- Crown participants / winners
