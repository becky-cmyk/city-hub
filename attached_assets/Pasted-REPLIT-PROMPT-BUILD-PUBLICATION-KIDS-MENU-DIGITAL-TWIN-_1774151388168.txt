REPLIT PROMPT — BUILD PUBLICATION + KIDS MENU DIGITAL TWIN SYSTEMS
WITH MASTER ADS MANAGER
PROJECT: CLT City Hub / City Hub
PRIORITY: Admin-first, template-based, QR-stable, archive-aware, sponsor-driven

==================================================
PRIMARY GOAL
==================================================

Build THREE connected admin/publishing systems inside the existing City Hub app:

1. MASTER ADS MANAGER
2. MICRO PUBLICATION DIGITAL TWIN SYSTEM
3. KIDS MENU GENERIC DIGITAL TWIN SYSTEM

Do NOT build the custom-back kids menu system yet.
That will be a later separate build.

The core requirement is that printed assets (publications and kids menus) each have stable QR URLs that do not change over time, while the online landing page updates to the newest version and visually shows older archived versions below.

==================================================
HIGH-LEVEL ARCHITECTURE
==================================================

MASTER ADS MANAGER is the source of truth for sponsorship/ad inventory and creative records.

PUBLICATION SYSTEM and KIDS MENU SYSTEM are separate systems that:
- use locked templates
- pull assigned sponsor/ad records from Ads Manager
- publish district-specific landing pages
- keep stable QR URLs
- show newest version first
- show older versions visually below

Use reusable components where appropriate, but keep the systems distinct in admin, data model, and workflow.

==================================================
SYSTEM 1 — MASTER ADS MANAGER
==================================================

Build a central Ads Manager that is NOT limited to publications and kids menus.

It is the future master inventory/assignment system for:
- Micro Publications
- Kids Menus
- Digital placements
- TV
- Radio
- Other sponsorship inventory later

For this first build, it must support the publication and kids menu systems directly, but the data model should be extensible for future media types.

----------------------------------
ADS MANAGER PURPOSE
----------------------------------

Ads Manager should function like a master sponsorship marketplace / inventory manager.

It must hold:
- sponsor/ad master records
- placement inventory
- creative assets
- assignment logic
- placement history
- optional pricing/package data if available
- ability to reuse the same sponsor across multiple districts/systems
- ability to assign one sponsor to all districts or selected districts
- ability to leave pricing blank if not available yet

It must be visual and organized by sponsorship/ad placement opportunities.

----------------------------------

----------------------------------

Holds the master sponsor/ad information.

Suggested fields:
- id
- sponsor_name

- category

- email
- phone
- website
- notes

- created_at
- updated_at

Suggested fields:
- id
- sponsor_id

- CTA text

- phone

- notes
- created_at
- updated_at

Represents every sponsorship/ad location available across systems.

Suggested fields:

- placement_name
- system_type (publication, kids_menu, tv, radio, digital, etc.)
- placement_group
- size_label
- visual_sort_order
- district_specific boolean

- active/inactive
- notes

Examples for first build:
- Publication Front Main
- Publication Front Back Placement
- Publication Small Placement

- Kids Menu Top Sponsor

D. Assignment Record
Connects sponsor/ad to placement + district + system + issue.

Suggested fields:

- start_date optional
- end_date optional
- status
- notes
- created_at
- updated_at

----------------------------------
ADS MANAGER REQUIRED FEATURES
----------------------------------

Build admin screens for:
- Sponsor list
- Sponsor detail/edit
- Creative assets manager
- Placement inventory manager
- Assignment manager

Need the ability to:
- create sponsor master record
- add one or more creatives
- assign sponsor to one district
- assign sponsor to selected districts
- assign sponsor to all districts
- assign sponsor to publications, kids menus, or both
- optionally leave pricing blank
- view where a sponsor is currently assigned
- view placement inventory visually
- later extend to TV/radio without rebuilding

Assignments should be manageable through checkbox, multi-select, or selection list UI.
Do not hardcode one-off sponsor placement flows inside publications or kids menus.

----------------------------------
PUBLICATION TEMPLATE
----------------------------------

Use a locked template, not a drag-and-drop layout editor.

Need a publication template system that:

- allows content and sponsor slots to be swapped

- assigned sponsors/ads
- previous issue if available

- “Generate full first draft from district + assigned sponsors + previous version”

You may optionally support later section-level regenerate, but the required first build is the full issue draft generation.
