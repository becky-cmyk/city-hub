--------------------------------------------------
PART 2 — PRIORITIZE GEO PRECISION
--------------------------------------------------

Geo precision is more important than perfect category precision.

The AI classification logic should prioritize detecting:
1. exact venue/location if possible
2. hub / micro-hub if resolvable
3. county
4. metro fallback

AI should look for location clues in:
- title
- summary
- body/article text
- source/domain patterns
- venue names
- known place/business/org names
- existing zone/location tags if already present

Rules:
- never leave geo empty if broader geography can be inferred
- preserve primary vs secondary geography
- if an article is about Gaston County but still relevant to the Charlotte metro, set:
  - primary = Gaston County
  - secondary = Charlotte metro
- do not collapse multi-geo relevance into one flat field

--------------------------------------------------
PART 3 — CATEGORY CLASSIFICATION
--------------------------------------------------

AI should classify content into:
- categoryCoreSlug
- categorySubSlug

Use the existing taxonomy and hierarchical category system that was just added.

Rules:
- if a subcategory is confidently matched, derive the correct core category as well
- do not assign only a subcategory without a core
- do not over-generic everything into “news”
- preserve future micro-category readiness
- if confidence is weak, assign the best core category and leave subcategory null rather than forcing a bad match

Examples:
- restaurant opening → core: food-dining, sub: restaurants
- brewery event → core: food-dining, sub: breweries or events depending on dominant intent
- high school football recap → core: sports, sub: local sports or high-school sports if taxonomy supports it
- city permit / rezoning / road closure → core: government-civic or community, not generic news
