You are implementing ONLY Phase 7A.

- Command Center exists

--------------------------------------------------
GOAL
--------------------------------------------------

Make Charlotte behave like an OPERATOR:

- can run tasks in the background
- continues working when UI is closed
- reports results when done

This fixes:
"Charlotte disappears when I close the box"

--------------------------------------------------
PART 1 — TASK SYSTEM (CORE)
--------------------------------------------------

Create a lightweight task system.

TABLE: tasks

Fields:
- id
- type (capture_process, outreach, proposal, followup, etc.)
- title
- payload (JSON)
- status:
  - pending
  - running
  - completed
  - failed
- progress (0–100, optional)
- result (JSON)
- error (optional)
- created_at
- updated_at
- completed_at
- source ("charlotte", "user", "system")

--------------------------------------------------
PART 2 — BACKGROUND EXECUTION
--------------------------------------------------

Implement a simple worker loop inside the app:

- picks up pending tasks
- marks as running
- executes logic
- updates progress if possible
- marks completed or failed

Keep it simple:
- interval-based worker (e.g., every few seconds)
- no external queues required

IMPORTANT:
Tasks must continue even if UI is closed.

--------------------------------------------------

- "generate proposals"

→ create a task

- task created
- status = pending/running
