Build v1 of a “Crown Program” inside the existing hub app.

GOAL
Create a working Year-1 micro-hub awards system that helps onboard businesses, creators, networking groups, and community organizations into the hub through a nomination + Hub Presence flow. This is NOT metro-wide yet. Year 1 is micro-hub only.

IMPORTANT V1 RULES
- Build only Micro Hub crowns in v1.
- Do NOT build Metro Crown yet.
- Keep admin control high and automation light.
- Make the system modular so Metro Crown can be added later.
- Keep the UI clean, simple, mobile-friendly, and operational.

CORE PROGRAM LOGIC

1) PARTICIPANT TYPES
Support these participant types:
- Business
- Creator
- Networking Group
- Community Organization

2) ELIGIBILITY
- Businesses must be locally owned and/or locally operated.
- Locally owned franchises may qualify.
- Corporate-owned chains do NOT qualify.
- Networking groups qualify based on local chapter activity, not national ownership.
- Chambers are excluded from v1 unless manually approved by admin later.

3) HUB RULE
- A participant can only compete in a hub where they have a real physical location, chapter, or regular meeting presence.
- They cannot compete in a hub where they do not operate.
- Multi-location businesses and multi-chapter organizations must be handled with parent/child structure:
  - Parent profile = master organization/business
  - Child profile/location/chapter = actual hub presence and actual competing entry
- Each child location/chapter competes only in its own hub.

4) YEAR-1 STATUS MODEL
Implement these statuses:
- Candidate
- Invited
- Verified Participant
- Nominee
- Qualified Nominee
- Crown Winner

Status intent:
- Candidate = AI-identified or community-submitted possibility
- Invited = outreach sent
- Verified Participant = accepted and activated paid Hub Presence
- Nominee = officially entered into category/hub
- Qualified Nominee = met threshold + participation rules
- Crown Winner = winner in that hub/category

5) PAYMENT / ENTRY FLOW
- If they proceed into the Crown nomination flow and purchase Hub Presence, they do NOT need the $1 verification flow separately.
- Hub Presence is required to participate in the Crown program.
- Price shown in v1:
  - Retail value: $699/year
  - Crown participant launch offer: $99/year
- Messaging must make clear:
  - they are NOT paying for votes
  - they are paying for Hub Presence, visibility, profile, and participation infrastructure

6) QUALIFICATION MODEL
Do NOT use one universal number for all categories.
Support category-level minimum vote thresholds.

Default v1 thresholds:
- High competition categories: 75 votes
- Mid competition categories: 40 votes
- Community / org categories: 25 votes

A nominee becomes Qualified Nominee only when:
- they have active paid Hub Presence
- they have a complete eligible profile
- they meet the category vote threshold
- if participant type is Networking Group or Community Organization, they must also have at least 1 active event/calendar entry

7) VOTING RULES
- One vote per user per category per hub
- Verified email required to vote
- User may vote in multiple hubs, but only once in each category/hub contest
- Allow suspicious vote flagging for admin review
- Build basic fraud flags:
  - unusual vote spikes
  - repeated IP patterns if available
  - duplicate-like voting patterns if detectable
- Admin can review and override if needed

8) HUB CATEGORY ENGINE
- Not all hubs run the same categories
- Admin can assign categories by hub
- Categories should be flexible per hub based on relevance and local density
- Build admin tools so categories can be turned on/off by hub
- This should be manual/admin-led in v1, but data model should allow future AI suggestions

9) YEAR-1 CORE LAUNCH CATEGORIES
Create category support for these initial categories:

Business / Venue
- Best Coffee
- Best Restaurant
- Best Brewery
- Best Food Truck
- Best Realtor
- Best Insurance Advisor
- Best Mortgage Broker
- Best Date Night Spot

Creator
- Best Local Creator
- Best Podcast

Networking / Community
- Best Networking Group
- Best Community Connector

10) EVENTS / CALENDAR
Minimal v1 event support only:
- Networking Groups and Community Organizations can create recurring or one-time events
- Events display on the hub calendar
- Events are linked to the organization/chapter hub presence
- Need only basic create/edit/list/display event support in v1
- Recurring meeting support is preferred if simple to build, otherwise basic recurring pattern placeholder is acceptable
