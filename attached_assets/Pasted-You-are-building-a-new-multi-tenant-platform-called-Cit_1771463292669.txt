You are building a new multi-tenant platform called "CityCoreHub" (master platform). It powers multiple city hubs (first city: CLT Connects). The architecture MUST support City → Zone → Content and Zone-based admins/operators. Public content must be SEO-indexable like a publication AND app-like on mobile. No WordPress. No plugin stacks.

STACK (required):
- Next.js (App Router) + TypeScript
- Tailwind CSS
- PostgreSQL
- Prisma ORM
- Zod validation
- Stripe (Checkout + Customer Portal + Webhooks) for billing/receipts (NOT net-30 invoicing)
- Resend-ready (email later) and Twilio-ready (SMS later) but implement stubs only if needed

CRITICAL GUARDRAILS:
- NO service worker in Phase 1 (avoid stale builds). Do not add next-pwa or SW libraries.
- HTML responses should be non-cacheable (no-store) to prevent stale shells.
- Public content must not be behind login. Admin only is gated.

ROUTING DECISION (LOCKED):
- City routing via PATH: citycorehub.com/[citySlug]/...
- Zones are NOT separate URL segments in Phase 1. Zone is required in data and used in filtering.
Example: /clt/directory?zone=south-end

PHASE 1 MVP (strong base + monetization):
A) Multi-tenant: City + Zone required for Businesses & Events
B) Public pages for: categories, zones, directory, events, articles, top lists
C) Global search + filters
D) Public submissions to moderation queue (never auto-publish)
E) Admin CMS (Super Admin + City Admin + Zone Admin)
F) Monetization fields + Stripe entitlements
G) Mobile-first “native feel” with device-based personalization (no login required)
H) Saved items + “Create a Plan” lists (device-based)
I) Share links for items (deep links)

ROLES / PERMISSIONS (server enforced):
- SUPER_ADMIN: you only; manage all cities/zones/users/content/entitlements
- CITY_ADMIN: manage content/submissions/sponsors within a city
- ZONE_ADMIN: manage content/submissions within assigned zones only
(EDITOR optional; if included, can edit but not publish without approval)
