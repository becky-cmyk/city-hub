2) EMAIL MERGE TAGS + COMPLIANCE
--------------------------------------------

2A) Supported merge tags (Phase 1)
- {{first_name}}
- {{city}}  (default "Charlotte")
- {{presence_name}}
- {{claim_link}}
- {{manage_link}}
- {{unsubscribe_link}}

2B) Marketing compliance rules
- All marketing templates MUST include:
  - physical footer block (brand/company + contact)
  - {{unsubscribe_link}}
- If missing at send-time, block send and show error.

2C) Unsubscribe behavior
- Generate unsubscribe URL per recipient (tokenized):
  /unsubscribe?email=...&token=...
- Hitting link creates email_unsubscribes(global_marketing) and suppresses future marketing sends.

2D) Suppression behavior
- Before enqueueing recipients, exclude:
  - email_unsubscribes (global_marketing) for marketing campaigns
  - email_suppression (bounce/complaint/manual) for all campaigns unless admin overrides (do NOT override in Phase 1)

3) RESEND INTEGRATION (SEND + WEBHOOKS)
--------------------------------------------

3A) Sending
- Use Resend API to send emails.
- Store provider_message_id per recipient for event correlation.

3B) Webhook endpoint
- Create endpoint to receive Resend events:
  - delivered
  - bounced
  - complaint
  - (open/click if supported/configured)
- Update email_campaign_recipients.status accordingly.
- If bounced => add to email_suppression(type=bounce)
- If complaint => add to email_suppression(type=complaint) AND add to email_unsubscribes(global_marketing)
