NOTE ON RSS/ICS DISCOVERY
Many sites don’t publish obvious RSS/ICS links. Build “feed discovery”:
- Fetch page HTML
- Auto-detect <link rel="alternate" type="application/rss+xml|application/atom+xml">
- Auto-detect iCal links (.ics) and “subscribe” buttons
If none found, use HTML_LIST parsing with a source-specific selector map.

NON-NEGOTIABLES
- No Google Places for events/jobs.
- No scraping restricted job boards (Indeed/LinkedIn/etc.).
- Keep it low cost: cache, rate-limit, incremental pulls, and disable sources by default until configured.
- Every ingested item must store provenance: source_type, source_url, external_id, raw payload reference.

CONNECTORS (implement)
A) RSS/ATOM CONNECTOR
/server/intelligence/connectors/rssAtomConnector.ts
- Input: feed_url
- Support conditional GET (ETag, If-Modified-Since) to reduce cost
- Parse RSS/Atom into normalized “items”
- Map:
  - events: if source configured as events feed
  - jobs: if source configured as jobs feed
- Upsert with (metro_id, source_type, external_id = guid/link hash)
- Store raw payload in source_raw_rows

B) ICS (iCal) CONNECTOR
/server/intelligence/connectors/icsConnector.ts
- Input: ics_url
- Parse VEVENTs
- External ID = UID + DTSTART
- Upsert events
- Store raw payload

C) EVENTBRITE CONNECTOR (Events)
/server/intelligence/connectors/eventbriteConnector.ts
- Requires secret EVENTBRITE_TOKEN
- Metro config includes:
  - location: {city, region/state, lat,lng,radius_miles}
  - categories (optional)
  - date range window (e.g. next 60 days)
- External ID = Eventbrite event id
- Upsert events
- Store raw payload
(Only Eventbrite-hosted events; that’s fine as a structured seed layer.)  [oai_citation:5‡Eventbrite](https://www.eventbrite.com/d/nc--charlotte--uptown/events/?utm_source=chatgpt.com)

D) USAJOBS CONNECTOR (Jobs)
/server/intelligence/connectors/usajobsConnector.ts
- Requires secrets:
  - USAJOBS_API_KEY
  - USAJOBS_USER_AGENT (email used in request header per docs)
- Use GET /api/Search with filters:
  - LocationName="Charlotte, North Carolina" (or metro config)
  - Radius / RemainInLocation, etc. per USAJOBS docs  [oai_citation:6‡developer.usajobs.gov](https://developer.usajobs.gov/api-reference/?utm_source=chatgpt.com)
- External ID = USAJOBS PositionID or AnnouncementNumber (prefer stable field)
- Upsert jobs
- Store raw payload

E) HTML_LIST CONNECTOR (Fallback for “no feed available”)
/server/intelligence/connectors/htmlListConnector.ts
Purpose: ingest event/job items from structured list pages that are stable enough to parse.
- Input: page_url + selector map in params_json:
  {
    "itemSelector": "...",
    "titleSelector": "...",
    "dateSelector": "...",
    "timeSelector": "...",
    "detailUrlSelector": "...",
    "venueSelector": "...",
    "descriptionSelector": "...",
    "externalIdStrategy": "detailUrl|hash(title+date+time)"
  }
- For each item:
  - Build external_id from detail URL if present; else hash(title+start_date+venue)
  - Parse dates/times with timezone = metro timezone
  - Upsert events/jobs
- Store raw HTML snapshot hash + extracted fields in source_raw_rows.payload_json
