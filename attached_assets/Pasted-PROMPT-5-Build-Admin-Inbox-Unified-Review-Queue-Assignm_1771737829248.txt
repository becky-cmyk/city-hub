--------------------------------------------
2) ITEM CREATION RULES (AUTO-GENERATION)
--------------------------------------------

Create inbox items automatically (system actor) when:

2A) Public submissions created
- For each submission form record, create an admin_inbox_item with item_type matching submission.
- Title format examples:
  - "New Business Submission: {name}"
  - "New Organization Submission: {name}"
  - "New Event Submission: {name}"
- Link route to the existing Submissions review screen.

2B) Presence claim confirm / transfer
- When claim confirmation initiated or transfer request created, create items:
  - presence_claim_confirm
  - presence_transfer_request
- Link to the admin presence record and transfer screen.

2C) CMS review queue
- When cms_content_items.status changes to "in_review", create:
  - cms_content_review
- Resolve automatically when published/archived OR when returned to draft.

2D) Vendors / Events needing approval
- If you have an approval gate (status draft->active), create:
  - vendor_review / event_review
- Only if status requires admin action.

2E) Billing alerts
- When a presence subscription becomes "past_due", create:
  - billing_past_due
- When founder grace window is within 7 days of expiry, create:
  - billing_founder_grace_expiring
Include due_at set to grace_end_at.

2F) Org supporter grace
- When supporter_grace_started_at is set, create:
  - org_supporter_grace_started
- When supporter_grace_end_at is within 7 days, create:
  - org_supporter_grace_expiring

2G) Email deliverability issues
- On Resend webhook:
  - bounce => create email_bounce_attention (unless already exists open for that email)
  - complaint => create email_complaint_attention (unless already exists open for that email)

Deduplication:
- For the same related_table+related_id+item_type, do not create duplicates if an OPEN item already exists.
- Instead, add a comment or update summary and log history.
