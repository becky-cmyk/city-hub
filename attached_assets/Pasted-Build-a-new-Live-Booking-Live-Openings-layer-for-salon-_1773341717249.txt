Build a new “Live Booking / Live Openings” layer for salon and service-based businesses inside the Hub platform.

IMPORTANT CONTEXT
This is NOT a full booking platform.
The Hub is the discovery + conversion layer.
The actual booking stays inside each provider’s existing booking app, booking link, or website.

We do NOT want to:
- manage calendars
- own appointment operations
- handle reschedules/cancellations
- process appointment payments
- become the system of record for bookings

We DO want to:
- show service providers on the Hub
- show services and pricing
- show “Available Today / Tomorrow” style openings
- let users feel like they can book directly from the Hub
- use embedded widgets where available
- use deep links / fallback booking buttons where needed
- track referral / booking clicks and contact actions
- support solo providers in salon suites, Phoenix-style suites, and other service businesses

TARGET BUSINESSES
Prioritize:
- salon suites
- independent stylists
- barbers
- nail techs
- estheticians
- lash artists
- massage therapists
- makeup artists
- wellness providers
- other service-based local businesses with appointment-driven revenue

KEY PRODUCT IDEA
The Hub should have a universal provider booking system built around provider profiles and booking module types.

Each provider can have one booking connection type:
1. embed_widget
2. popup_widget
3. deep_link
4. call_text_fallback
5. manual_live_opening
6. api_connected (future placeholder only)

Do NOT force all providers into one workflow.

CORE EXPERIENCE
A user should be able to:
- discover a provider by category, specialty, or neighborhood
- view provider profile
- see services and prices
- see live openings like “Available Today” or “Tomorrow”
- click Book Now, Call, Text, or Instagram
- complete the actual booking in the provider’s own system

The Hub should feel like the front door, not just a directory.
