# Venue Channel + Social Selling Backbone

### For City Metro Hub Platform

## 1. Core Concept

Each **business can have a Venue Channel** inside the Hub.

The Venue Channel is the business’s:

* media channel
* content hub
* live stream location
* venue screen feed
* social selling entry point

The Hub distributes that content across:

* venue screens
* venue channel pages
* micro hubs
* categories
* pulse feeds
* metro channel feeds

---

```text
City
```

# 4. YouTube Integration

YouTube is used only for **video hosting and streaming**.

### Structure

Use **one YouTube channel per city**.

Example:

```text
CLT Metro Hub YouTube
```

Inside that channel create playlists:

```text
Metro Channel
Venue Channels
Neighborhood Channels
Live Replays
Categories
```

### Video workflow

```text
Content created
↓
Uploaded to YouTube
↓
Added to playlist
↓
YouTube URL stored in Hub database
↓
Hub embeds video
```

# 6. Live Streaming

Live streams use **YouTube Live**.

### LiveSession fields

```text
id
venueId
cityId
youtubeLiveUrl
startTime
endTime
status
attachedOfferIds
```

### Flow

```text
Venue streams to YouTube
↓
Hub detects active live session
↓
Venue Channel switches to Live Mode
↓
Hub Live feed displays stream
↓
Venue screens display stream
```

After stream ends:

```text
YouTube saves replay
Hub returns to programming loop
```

# 12. QR System

QR codes should link to Hub routes, not YouTube.

Example:

```text
/hub/offer/{offerId}
/hub/live/{liveSessionId}
/venue/{slug}
```

This keeps traffic inside the Hub.
